Railway Cron Configuration
==========================

Two operational buckets — use separate Railway cron services (or schedules):

--- Stable cron (main) ---
Start Command:
python scripts/cron/cron_run_scheduled_scrapers.py

(Legacy shim still works: python scripts/cron_run_scheduled_scrapers.py)

Suggested schedule:
25 20 * * 1    (Monday 8:25 PM UTC ≈ 3:25 PM ET EST)

Runs: NGA, SAAM, African Art, Eventbrite embassies/extras, Meetup, Wharf DC,
Webster's, ACF DC, NYC/SF/LA standalone scrapers, seasonal DC Parade / Tulip Day, etc.

Log file: logs/cron_stable_YYYYMMDD.log

--- Protected cron (403 / Cloudflare / proxy-sensitive) ---
Start Command:
python scripts/cron/cron_run_protected_scrapers.py

(Legacy shim still works: python scripts/cron_run_protected_scrapers.py)

Suggested schedule (offset from stable, e.g. 30 min later):
55 20 * * 1    (Monday 8:55 PM UTC)

Runs: Smithsonian Asian Art, National Portrait Gallery, Hirshhorn only.

Log file: logs/cron_protected_YYYYMMDD.log

Bucket classification: scripts/cron/cron_bucket_config.py
(Shim: scripts/cron_bucket_config.py)

Setup Steps:
1. Railway Dashboard → Project → add or duplicate cron service per bucket
2. Set Start Command and Cron Schedule for each
3. Save

Environment Variables Needed:
- DATABASE_URL (auto-provided if using Railway Postgres)
- FLASK_ENV=production
- SCRAPER_PROXY_* / Webshare vars if using proxy for protected scrapers
- Any API keys (GROQ_API_KEY, etc.)
